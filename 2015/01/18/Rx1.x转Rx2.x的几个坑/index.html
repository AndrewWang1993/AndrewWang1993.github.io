Rx1.x转Rx2.x的几个坑

Jan 18, 2015 in Android

使用Rxjava1.x已经很久了，Rx项目组可能觉得初期架构有问题又发展个2.x版本.我最近抽时间把1.x换成了2.x，实在不能换的就进行了转换或者直接自己实现。

下面就简单概括概况两者的区别和怎么平滑升级。

两者的gradle库

概述

  • 两者可以共存
  • 第二个版本包含第一个版本的部分功能

区别

  • RxJava2.x不允许空值发射

    RxJava2 no longer permits emitting a null value from a producer. Emitting a null value from an observable, flowable, or subject will throw a NullPointerException:

    被观察者不允许发射Null，本意是好的，但是如果将网络回调的数据通过Observeable进行发射，难免会有NULL，此时可以进行定制操作符,删除库里判空的语句。

  • Subscriber 变成了 Disposable

    因为和Reactive-Streams里命名冲突，所以Subscriber改名字啦，于是原来的.unsubscribe() 来解绑也变成了.dispose()来释放资源

  • CompositeSubscriber 变成了 CompositeDisposable

    RxJava1

    RxJava2

  • BackPressuer背压方式

    在Rxjava2.x中背压方式的实现由一个新类来实现——Flowable类，它能够处理被观察者发送速度比观察者快的情况。
    新的RxJava2.x所以操作符都已经支持了背压功能，但是这仅仅意味着消费者或者生产者不会抛出MissingBackpressureException exception，如果生产太快这个异常就会从操作符(onNext)抛出。
